## Version History

### Version Numbering

We use [Semantic Versioning](https://semver.org/):

- **MAJOR** version (1.x.x) - Incompatible API changes
- **MINOR** version (x. 1.x) - New features, backwards compatible
- **PATCH** version (x.x.1) - Bug fixes, backwards compatible

### How to Update

1. **Check for updates:**

   - Email notifications (if subscribed)
   - Check Gumroad product page
   - Watch GitHub repository

2. **Download latest version:**

   - Log in to Gumroad
   - Go to Library
   - Download latest files

3. **Update your project:**
   - Backup your `src/config.js`
   - Replace files with new version
   - Restore your config
   - Test locally
   - Deploy

### Staying Updated

**Automatic notifications:**

- Gumroad emails you when files are updated
- Star the GitHub repo for release notifications

**Manual check:**

- Compare your CHANGELOG.md version number
- Latest is always on Gumroad product page
